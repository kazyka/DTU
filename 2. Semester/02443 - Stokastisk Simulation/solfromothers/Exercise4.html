Exercise 4¶

  • Write a discrete event simulation program for a blocking system, i.e. a system with n service units and no waiting room.
  • The arrival process is modelled as a Poisson process.
  • Choose first the service time distribution as exponential.
  • Record the fraction of blocked customers, and a confidence interval for this fraction.
  • The programme should take the offered traffic and the number of service units as input parameters.

Example: n = 10, mean service time = 8 time units, mean time between customers = 1 time unit (corresponding to an offered traffic of 8 erlang), 10 x 10.000 customers.

First the eventlist of the simulation is set up as a class. There are two possible events: Arrival of a customer (arrive) and departure of a customer (depart). Whenever a customer arrives, another arriving customer is generated in the eventlist along with its corresponding arrival time based on the given probability distribution. It is then checked whether a service unit is available. If this is the case, a departing customer is generated in the eventlist along with its corresponding departure time, also based on the given probability distribution. If no service unit is available, the customer is rejected and stored as such. This continues until N=10.000 arriving customers have been generated.

Confidence limits based on sub-sample¶

  • We want to estimate some quantity θ
  • We obtain n different (independent) estimates $\hat{\theta}_i$ .
  • The central limit theorem motivates us to construct the following confidence interval: \begin{equation} \bar{\theta}=\frac{\sum_{i=1}^n\hat{\theta}_i}{n}\quad s_\theta^2= \frac{1}{n-1}\left(\sum_{i=1}^n\hat{\theta}_i^2-n\bar{\theta}_i^2\right) \end{equation}

We thus generate 10 instances of 10.000 customers visiting to obtain $\bar{\theta}$ and the standard deviation $s_\theta$

In [6]:
np.random_intel.seed(1)
rand = np.random_intel.randint(0,10000,size=10)
rarr=[]
for i in rand:
    _, r,_,_,_ =blockingSystem(n,N,timeService,timeCustomer,seed=i)
    rarr.append(r)
res = np.array(rarr)/N*100
theta = res.mean()
S_theta = 1/(n-1)*(sum([i**2 for i in res])-n*theta**2)

print("theta = {:0.4f}".format(theta))
print("s_theta = {:0.4f}".format(S_theta**(1/2)))

theta = 12.3370
s_theta = 0.3531

This allows us to calculate the confidence interval based on: \begin{equation} \left[\bar{\theta}+\frac{S_\theta}{\sqrt{n}}t_{\frac{\alpha}{2}}(n-1);\bar{\theta}+\frac{S_\theta}{\sqrt{n}}t_{1-\frac{\alpha}{2}}(n-1)\right] \end{equation} with $\alpha = 0.05$

Hyper-Exponential Distribution¶

The hyper-exponential distribution generates numbers based on two different exponential distributions with different probabilities of generating from each. Here, \begin{equation} p_1 = 0.8, \lambda_1 = 0.8333, p_2 = 0.2, \lambda_2 = 5.0 \end{equation} Below, the hyper-exponential distribution is defined as a function.

In [10]:
p = [0.8,0.2]
l = [0.8333,5]

def hyper():
    i = np.random_intel.rand()
    if i<p[0]:
        return np.random_intel.exponential(l[0])
    else:
        return np.random_intel.exponential(l[1])

Theoretical Service time¶

  • With arrival intensity $\lambda$ and mean service time $s$
  • Define $A=\lambda s$ \begin{equation} B = P(n) = \frac{\frac{A^n}{n!}}{\sum_{i=0}^n\frac{A^i}{i!}} \end{equation}
  • Valid for all service time distributions
  • But arrival process has to be a Poisson process

Comparing this with the result from the first part of this exercise can hint the validity of the model
